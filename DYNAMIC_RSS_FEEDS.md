# Dynamic RSS Feed Generation

## Overview

The RSS feeds are now dynamically generated by Hugo instead of being hard-coded static files. This ensures that the feeds always reflect the latest content without manual updates.

## Changes Made

### 1. Created Hugo RSS Templates

- **`layouts/bulletins/list.rss.xml`**: Generates the main security bulletins RSS feed
  - Output location: `/bulletins/feed.xml`
  - Replaces: `/static/adobe-security.xml`
  - Includes: All security bulletins (APSB entries) sorted by date
  - Filters posts by looking for "APSB" in the title

- **`layouts/products/single.rss.xml`**: Generates product-specific RSS feeds
  - Output location: `/products/{product-name}/feed.xml`
  - Replaces: All files in `/static/feeds/{product-name}.xml`
  - Includes: Bulletins filtered by product category
  - Dynamically filters based on product slug

### 2. Updated Hugo Configuration

Modified `config.json` to enable RSS output for:
- **Section pages**: Added RSS to `outputs.section` (for section list pages)
- **Product pages**: Added RSS to `outputs.page` (for individual product pages)

Note: The `bulletins` content type is treated as a section, and the output is configured via the section outputs.

### 3. Updated Feed URLs in Templates

Updated references in the following templates to use Hugo-generated feeds:
- `layouts/index.html`: Changed `/adobe-security.xml` → `/bulletins/feed.xml`
- `layouts/products/list.html`: Updated product feed links to use `{product-page}/feed.xml`
- `layouts/products/single.html`: Updated product RSS buttons to use relative `feed.xml`
- `layouts/feeds/list.html`: Updated bulletins feed link to `/bulletins/feed.xml`

## Testing

After rebuilding the site with `hugo`, verify:

1. **Main bulletins feed**: Visit `/bulletins/feed.xml` and confirm it contains current bulletins
2. **Product feeds**: Visit `/products/{product-name}/feed.xml` for each product
3. **Feed content**: Verify each feed includes:
   - Proper XML structure
   - Current publication dates
   - Correct bulletin titles and links
   - Product categorization

### Cleanup Commands

```bash

# Or remove the entire feeds directory if it only contains XML files
rm -rf /workspaces/adobe-digest/static/feeds/
```

## Benefits

1. **Automatic Updates**: Feeds update automatically when content changes
2. **Consistency**: Feed content is always in sync with the website
3. **Maintainability**: No need to manually update feed files
4. **Product Filtering**: Each product page automatically generates its own feed
5. **Date Accuracy**: Publication dates always reflect the latest build time

## Notes

- The feeds use the same RSS 2.0 format as the previous static files
- Hugo rebuilds feeds automatically during each site build
- Feed URLs have changed slightly (e.g., `/adobe-security.xml` → `/bulletins/feed.xml`)
- Old feed URLs should redirect or be removed to avoid confusion
